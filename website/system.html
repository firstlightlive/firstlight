SECTION 04

THE JUDGE

The system doesn't just track. It punishes. Automatically. Publicly. Permanently.

STEP 1 — RULE BROKEN
The system detects the violation.
When a rule is broken, the admin logs a slip. The moment it's submitted — the system takes over. The admin has no more control.
STEP 2 — PUNISHMENT AUTO-ASSIGNED
The system decides the penalty. Not the admin.
Standard violation: 20km walk with GPS proof. Fortress breach: 50km walk + 5km run OR 100km cycling. The code assigns it. No negotiation. No appeal.
STEP 3 — PUBLISHED INSTANTLY
The slip goes live on the public accountability ledger — instantly.
Date. Rule broken. Penalty status: RED — PENDING. Anyone on the internet can see it. The admin cannot hide it, delete it, or make it private. The database trigger rejects the attempt.
STEP 4 — STRAVA GPS VERIFICATION
The server reads GPS distance directly from Strava. Not from the admin.
The admin selects a real Strava Walk or Ride activity. The database trigger then reads the actual distance from the Strava table — not from the browser. The admin can't type a fake number. Cycling counts at 0.5x (need double). The activity must have happened after the slip date — no old walks. Each activity can only be used for one penalty.
STEP 5 — ESCALATION ENGINE
7 days to complete. Then it gets worse. Every single day.
After a 7-day grace period, the penalty escalates by +3 km per day. Day 8: 23 km. Day 15: 44 km. Caps at 70 km — then a new cascade penalty spawns at 25 km, which also escalates. The chain never ends until every penalty is walked. The escalation is computed by the database server — not the browser. Changing your device clock does nothing.
PERMANENT RECORD
Cleared ≠ Erased. The scar remains. Forever.
Every slip — the date, the rule, the penalty, the Strava proof — lives permanently on the public accountability ledger. 10 years from now, you'll still see it. The proof URL is write-once — can never be changed. The status is one-way — CLEARED can never revert to PENDING. That's the point. Consequences should leave marks.
The admin built the system.
The system now controls the admin.
He can't delete slips. He can't fake distance. He can't use old walks. He can't change proof after it's set.
He can only do one thing — walk the punishment and let the GPS prove it.
